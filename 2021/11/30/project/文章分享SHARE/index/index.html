项目简介

SHARE 文章分享系统是一个创作者分享文章的管理系统，主要为各类创作者提供友好的创作交流平台。继承了传统博客的形式，注重内容的呈现和用户兴趣驱动，拥有着一定的交流性和传播性，可以实现创作者之间的互动，例如对作品的评论、收藏和转发等。在本系统中，创作者将通过文章的形式分享作品及其创作经验，展现出一种“分享+社交”的模式。

框架使用

Vue是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

使用 vue 的主要原因是我目前还只是会 vue，并且 vue 相比于其他框架来说，对于我这种菜鸟相对来说要简单一点，虽然有点忘记了 vue，但最近重新系统学习了 vue 相关的知识，当然也只是大概过一过，但也算是捡回了记忆。

组件库使用

这次因为是主打方向是 web 端，也就没有使用上个项目的vant组件库，而是用的vuetify来作为组件库开发，也没有使用element-UI，因为我感觉 element 其实更适合去做后台管理系统，而且我也更喜欢 vuetify 的主题颜色。

插件使用

vue-router

Vue Router是 Vue.js 官方的路由管理器。它和 Vue.js 的核心深度集成，让构建单页面应用变得易如反掌。包含的功能有：

  • 嵌套的路由/视图表
  • 模块化的、基于组件的路由配置
  • 路由参数、查询、通配符
  • 基于 Vue.js 过渡系统的视图过渡效果
  • 细粒度的导航控制
  • 带有自动激活的 CSS class 的链接
  • HTML5 历史模式或 hash 模式
  • 自定义的滚动条行为

上面这段话是从官方 copy 下来的，其中我觉得主要或者常用到也就是路由配置、路由参数、导航控制、history、css 激活。当然本项目中没有使用后三个。

Vuex

首先不谈 Vuex，人们是如何实现组件通信的？

其实方法有很多，这里只介绍几个比较简单的方法即让所有组件都能访问到，但要如何让所有组件都能访问到？

  1. 子组件通过事件将数据传给父组件，父组件再通过 props 向目标子组件传递数据。
  2. 创建一个空 Vue 实例，用它来触发事件和监听事件,巧妙地实现了任何组件间的通信。
  3. 使用 localStorage 来存储子组件要传的数据，目标组件再从 localStorage 中取得。

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

Vue CLI

Vue CLI是一个基于 Vue.js 进行快速开发的完整系统，提供：

  • 通过 @vue/cli 实现的交互式的项目脚手架。
  • 通过 @vue/cli + @vue/cli-service-global 实现的零配置原型开发。
  • 一个运行时依赖 (@vue/cli-service)，该依赖：
    • 可升级；
    • 基于webpack构建，并带有合理的默认配置；
    • 可以通过项目内的配置文件进行配置；
    • 可以通过插件进行扩展。
  • 一个丰富的官方插件集合，集成了前端生态中最好的工具。
  • 一套完全图形化的创建和管理 Vue.js 项目的用户界面。

Vue CLI 致力于将 Vue 生态中的工具基础标准化。它确保了各种构建工具能够基于智能的默认配置即可平稳衔接，这样你可以专注在撰写应用上，而不必花好几天去纠结配置的问题。与此同时，它也为每个工具提供了调整配置的灵活性，无需 eject。

当然现在 Vue 又出了Vite这个脚手架，但是这个脚手架因为是刚出来，虽然颠覆 webpack 的构建模式，但是时间太短了，有待时间检验。

所以本项目还是使用官方的 Vue CLI 来构建项目。

vue2-editor

vue2-editor是本项目所使用的富文本编辑器，可以支持 markdown、视频、图片等，可以方便创作者直接编写文章。

toast_demo_based_vue

因为自己做的项目用到的是 Vuetify，然而它本身不支持全局的消息提示，所以参考了网上的教程之后，使用了一个第三方 Vue 的全局 Toast 消息插件toast_demo_based_vue。

axios

本项目使用的是比较为流行网络请求插件axios，其实 Vue 自己也有一个网络请求库vue-resource，但官方自己就推荐使用 axios 来发起请求，并且是方法大致相同，所以也就没有使用这个插件。

后端模块

  • urls.py 处理前端和后端交互的接口。
  • views.py 处理前端的请求事务。
  • models.py 用来连接操作数据库。
  • settings.py 用来配置数据库信息。

数据库设计

  • backend_creater 表负责存储创作者的信息

  • backend_category 表用于文章分类筛选

  • backend_article 表用于存储创作者的文章信息

  • backend_comment 表负责存储用户的文章评论
