|字数总计:1.8k|阅读时长:5分钟|阅读量:

项目介绍

这是创新实践学习的一个小项目，其实我前年就开始接触微信小程序，但那时候也只是稍微了解了一下微信小程序，跟着他们前端混了一段时间，也没有怎么系统得学习小程序，所以这次做这个微信小程序也是做的很简单的、入门的一个todoList项目。请各位大佬不要嫌弃我是个菜逼。

MyToDo微信小程序

界面展示


这三幅图片是MyToDo小程序的三个主页面，其实从界面可以看得出来很明显这个小程序项目是很简单的，没有什么很特别的东西，就是和那两个视频写的功能差不多，只不过添加了一个云开发而已，后面会介绍云开发的过程的。

  • 首页就是来展示当前有哪些任务，即人物列表，可以左右滑动，表示删除和已完成。
  • 添加任务这个页面，就是可以将日常的一些事情记录下来，有事项标题、紧急度、内容以及选择截止时间，最后确定保存，跳转到任务详情页面。
  • 个人中心页面可以看到当前的任务数量，以及可以用户登录，之后就是一些填充的功能，比如分享小程序、意见反馈、常见问题啊。

虽然页面比较偏简约风格，但功能实现也还是全的。

组件介绍

本项目主要用到的组件就是Vant这个移动端组件库，主要用来构成按钮、任务滑块、日期和时间选择器、tab-bar、文本框等等。

Vant 是有赞前端团队开源的移动端组件库，于 2017 年开源，已持续维护 4 年时间。Vant 对内承载了有赞所有核心业务，对外服务十多万开发者，是业界主流的移动端组件库之一。目前 Vant 官方提供了 Vue 版本和微信小程序版本，并由社区团队维护 React 版本。

可能有很多人没听说过有赞这个团队，但如果经常混React论坛，会经常用到他们的产品，比如zent(React组件库)、SuperMan(跨项目通用的打包工具)。感兴趣的可以了解、使用一下。

插件介绍

Day.js

用这个插件是为了对时间格式化，就是用户根据时间选择器选择时间，然后解析，以及保存当前系统时间和计时。然后还有另一个跟时间相关的插件，就是moment.js

两者几乎是一模一样，使用方法也大致差不多，但day.js只有几十kb，而moment有几百kb，所以基本上都推荐使用day.js。

子组件通信

子组件通信其实在微信小程序还没有更新的时候，基本上都是用的插件来实现的，也就是pubsub.js这个插件，当然它也不只是用于小程序，还有其他的一些框架，比如vue、react，就是说只要有组件通信或者订阅发布的需求，都可以用这个插件，因为它本就是用来实现订阅和发布消息的，只不过可以被用来间接实现子组件通信。

但现在微信小程序官方已经支持了子组件通信方法，就是用EventChannel这个API，可以简化使用子组件通信。

小程序网络请求函数

微信小程序网路请求函数是官方自己根据ES8的基于promise的http网路请求库。

export const request = (params) => {
    // 判断 url中是否带有 /my/ 请求的是私有的路径 带上header token
    let header = {
        ...params.header
    };
    if (params.url.includes("/my/")) {
        // 拼接header 带上token
        header["Authorization"] = wx.getStorageSync("token");
    }
    wx.showLoading({
        title: "加载中",
        mask: true
    });
    // 定义公共的url
    const baseUrl = "https://api-hmugo-web.itheima.net/api/public/v1";
    return new Promise((resolve, reject) => {
        wx.request({
            ...params,
            header: header,
            url: baseUrl + params.url,
            success: (result) => {
                resolve(result.data.message);
            },
            fail: (err) => {
                reject(err);
            },
            //无论如何都执行
            complete: () => {
                    wx.hideLoading();
                }
            }
        });
    })
}

但是我是用云函数来发送http网络请求的，原理是一样的，都是基于promise来发送网络请求的。

微信小程序云开发

微信小程序的云开发，其实在我看来，和已经集成好的后端是差不多的，比如微信云开发的云函数，其实就相当于后端的API接口，当我们在调用云函数的时候就相当于调用接口提供的服务，只不过不是restful风格，而是以函数包装ip地址来调用的，其实际上还是访问你所需要的云函数api。而云开发所提供的数据库就相当于后端的数据库，但还是有点区别的：

后端在初始化数据库的时候需要指明表中的属性名，而云开发的数据库则是不需要的，是由用户存入的数据来指定的，也就是说一张表可以有不同属性的字段。

如何开始云开发

  1. 在project.config.json中指明小程序和云函数的根目录。
  2. 在index.js中编写函数，注意云环境一定要先初始化。
  3. 在config.json中指明要调用的另外的函数api和触发器。
  4. 在package.json中指明要引用的第三方包。
  5. 然后上传部署就ok了。

数据库的话就需要先获取数据库对象以及相应的表。然后就正常的数据库函数调用了。

消息订阅

这里的消息订阅并不是前面所说的那个pubsub的消息订阅，而是微信小程序的消息订阅，通过模板消息来通知用户时间快截止了。

其实微信小程序提供的消息订阅功能是已经包装好了的，很容易上手，不像后端那样麻烦，还要各种配置。

消息订阅就是发布方给订阅方发布消息，如果看过第二个视频的话，就很容易理解这个概念，那个视频是通过pubsub.js来实现页面通信，但那其实是消息订阅来变通实现的，现在微信小程序提供EventChannel来页面间事件通信通道。

但消息订阅这个其实在后端用得会更多一些，其实也就是中间件，常见的消息中间件有rabbit、kafka。

小程序学习总结

我小程序学习是跟着B站视频，就是黑马商场和硅谷云音乐，就是先跟着那种项目实战（包括讲解）来敲一遍，虽然不是很懂，但至少对整个技术架构有了大致的了解，然后再来自己通过官方文档和查阅资料来实现一个适合自己的项目，因为在这个阶段肯定会碰到很多bug，当你去解决这些bug的时候，就是对技术更加深入的了解，最后就可以考虑去静下心来去读一读源码，来了解透这个技术，甚至可以去优化或者重构一下这个框架。
